코드 품질을 재평가해 `docs/QUALITY_SCORE.md`를 갱신한다. pr-create가 PR 전에 자동 호출하는 게이트다.

## 평가 기준

`docs/swift-code-quality.md`의 4축 + 보안 + 디자인 충실도로 등급(A~F)을 매긴다:

- **가독성** — 이름·구조·흐름이 한눈에 읽히는가.
- **예측성** — 함수가 시그니처대로 동작하고 부작용이 숨지 않는가.
- **응집도** — 함께 바뀌는 것이 함께 모여 있는가.
- **결합도** — 레이어 경계(UI 표시 / Logic 값 처리)가 지켜지는가.
- **보안(시크릿 격리)** — 시크릿 평문 누출 0. 누출 발견 시 **BLOCK**.
- **디자인 충실도** — `docs/references/Design.md`와 대조.
- **빌드/lint/테스트** — green 필수.

## 절차

1. 변경 범위를 파악한다(`git diff`). 전면 재평가가 아니라 바뀐 부분 중심으로 본다.
2. 각 축을 명세 대조로 채점한다(주관 취향 금지).
3. `docs/QUALITY_SCORE.md`의 등급표와 이력을 갱신한다(날짜·요약 포함).
4. **기준 미달**(보안 BLOCK, 또는 핵심 축 D 이하)이면 PR 전 개선 항목을 구체적으로 권고하고 진행을 보류 권유한다.

`QUALITY_SCORE.md`는 동기화 대상이 아니다 — 이 커맨드만 갱신한다.
