작업 후 남은 부산물을 정리해 워킹 트리를 리뷰 가능한 상태로 되돌린다. **소스·문서 본체는 건드리지 않고** 임시 산출물만 제거한다.

## 정리 대상

1. **머지된 exec-plan** — `docs/exec-plans/`에서 이미 머지된 작업의 계획 파일이 남아 있는지 확인하고 삭제한다. (merge 스킬이 지우는 게 정상이나 누락분을 회수.) 진행 중인 작업의 exec-plan은 **삭제하지 않는다** — 머지 여부가 불확실하면 삭제 전 확인한다.
2. **임시 작업 디렉터리** — `_workspace/`(QA 리포트 등) 잔여물.
3. **`/tmp` 산출물** — `/tmp/view-inventory.new` 등 훅·스크립트가 남긴 임시 파일.
4. **빌드 산출물** — `DerivedData/`, `build/`, `*.xcresult` 등 git 비추적 빌드 결과물.

## 절차

1. `git status`로 비추적·변경 파일을 파악한다.
2. 위 대상만 골라 삭제하고, 무엇을 지웠는지 목록으로 보고한다.
3. 삭제 후 `git status`가 의도한 변경만 남았는지 확인한다.
4. 소스·확정 문서·진행 중 exec-plan이 실수로 지워지지 않았는지 재확인한다.

`.gitignore`에 잡히는 비추적 빌드물은 자유롭게 지우되, **추적 중인 파일 삭제는 반드시 보고**한다.
