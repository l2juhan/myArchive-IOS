# core-beliefs — 프로젝트 설계 철학

스킬·에이전트가 판단이 필요할 때 참조하는 설계 원칙. PRD/Design.md가 *무엇/어떻게*라면, 이 문서는 *왜 그렇게 결정하는가*다.

## 1. 속도가 곧 가치다
조회 속도와 검색 편의가 최우선(PRD 3, 6.1). 콜드 스타트→목록 1초 이내, 검색 즉시 반영. 기능을 더할 때 "이게 빠른 접근을 해치는가?"를 먼저 묻는다. 잠금을 기본 꺼짐으로 둔 것도 이 원칙의 산물.

## 2. 저장 보호는 항상, 진입 잠금은 선택
Keychain 암호화(저장 보호)와 앱 진입 잠금(인증)은 **별개 레이어**다(PRD 7). 저장 보호는 끌 수 없는 기본값, 진입 잠금만 사용자 선택. "잠금을 꺼도 평문 저장은 절대 없다"가 메모장 대비 핵심 개선점.

## 3. 필요한 값만, 안전하게
시크릿은 기본 마스킹(블러), 터치 단방향 해제(F-3). 복사는 필드별 개별(F-4) + 클립보드 만료(F-8). 묶음 노출/복사를 만들지 않는다. 어깨너머 엿보기를 상수로 가정한다.

## 4. 검증된 OS API에 기댄다
1인 개발의 보안 검증 한계를 인정하고, 자체 암호화 대신 Keychain·LocalAuthentication 같은 검증된 표준에 의존한다(PRD 7, 13). 외부 SDK·네트워크 의존성은 0을 지향한다(완전 오프라인).

## 5. 디자인은 재현하되 이식하지 않는다
HTML/CSS를 옮기지 않고 Design.md의 확정값을 SwiftUI 관용구로 픽셀 단위 재현한다(고충실도). 토큰을 단일 소스로 두고 하드코딩을 금지한다.

## 6. 경계를 지킨다
UI는 표시, Logic은 값 처리. 시크릿은 UI 레이어를 평문으로 통과하지 않는다. 경계면(메타↔Keychain↔UI) 정합성은 QA가 교차 검증한다 — 경계면 버그는 한쪽만 고치면 재발한다.

## 7. 학습 프로젝트임을 존중한다
개발자가 곧 실사용자(PRD 3). 마일스톤 M1~M6은 학습 곡선(SwiftUI→SwiftData→Keychain→LocalAuthentication→보안 마감→폴리싱)과 묶여 있다. 과도한 추상화보다 명확하고 따라가기 쉬운 코드를 선호한다.
